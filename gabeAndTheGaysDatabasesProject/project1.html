  1. What command can you type into the terminal to check if apache is currently running?

    2. service apache2 status

  2. What is the difference between ServerRoot and DocumentRoot?

    3. DocumentRoot is path to the files deliverable by the server. ServerRoot is path to executable in the server directory.

  3. What port does your web server listen to for HTTP connections from browsers? Can you change this port?

    4. HTTP is port 80. Change this port with "Listen <new port>" in the ports.conf file.

  4. In what directory do you need to place all of your HTML and PHP documents for apache to serve them up? Test by navigating to this directory, and if your hunch is right, you should find some It Worked! HTML document.

    5. HTML and PHP files should be put in the /var/www/html/ directory.

  5. What file contains all of Apache's traffic logs? What about its error logs?

    6. Traffic logs are in /var/log/apache2/access.log. Error logs are in /var/log/apache2/error.log.

  6. What is a directory index file? Why would it be nice to have one in each directory?

    7. A directory index file is an html file (usually called index.html) that contains a list of filenames. DirectoryIndex sets a list of resources to look for when the client requests the directory index. This can be used to navigate through a site.

  7. How do you give every user on your Linux machine their own web space? Look into public_html?

    8. Public_html is the directory where all users can put their html files so all clients can have access to the files. These pages can generally be accessed with http://your_IP_addr/~username

  8. How do you create a password protected directory (like I did for the class notes)? Look into .htpasswd and .htaccess files.

    9. The .htaccess file provides a directory at root level that requires valid user info to be accessed. The .htpasswd file contains usernames and passwords for the page or directory.

  9. When the browser tries to access a page that doesn’t exist, the HTTP protocol issues a 404 error code. There’s a way to send the browser to a particular document upon issuing this error. How?

    10. You can use a page redirect to send the browser to a particular page after a 404 error with <meta HTTP-EQUIV="Refresh" CONTENT="5; URL=otherPage.htm">.

  10. Apache implements the HTTP protocol. The protocol is extremely simple, with just a few commands. What is the difference between the GET, POST, and HEAD commands?

    11. GET: used to retrieve data

    POST: used to send data to the server - uses HTML forms

    HEAD: like GET, but only sends the status line and header data
